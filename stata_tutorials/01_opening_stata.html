Opening Stata for the First Time

For best viewing experience, watch this video in fullscreen mode.

This initial video is intended to provide a general orientation to Stata for those who have never used it. I recommend you watch with Stata open and go through each of the steps.

  1. When you open Stata, it will by default have a white background with black text. I recommend changing to dark mode as it provides higher contrast and may be easier on your eyes. To change to dark mode, go to Edit (menu bar, top left) →\rightarrow→ Preferences →\rightarrow→ General preferences... →\rightarrow→ select the Overall color scheme dropdown and set to Dark. Select Okay. You can see there other ways to customize the display in Stata to suit your preferences.

  2. Results window

    • Current version, license information
    • Clear results window:
      cls

  1. Command window / command line
    • Set current directory
      • Check current directory (also in lower left of Stata window)
        pwd
      • Change to new directory:
        cd "C:\Users\michael.baker\Documents\ss368"
    • For the remainder of this training, I'm using one of the native Stata datasets for training purposes. However, I'll be loading this dataset from my local computer as that is what we will typically be doing. To follow along, you can open and save this dataset to your computer:
      sysuse auto, clear 
save "auto.dta"
      Now just replace the filepaths below with your own filepaths.
    • Load a dataset
      use "C:\Users\michael.baker\Documents\ss368\raw_dta\auto.dta", clear
    • In general, you will always use the , clear option. If you have data loaded into Stata and then try to import data without , clear you will get an error. The , clear option is equivalent to:
      clear 
use "C:\Users\michael.baker\Documents\ss368\raw_dta\auto.dta"
    • Absolute versus relative file paths. The preceding example using an absolute file path. It includes the entire path to the .dta file. Alternatively, we could use a relative file path that takes into account the current directory.
      use "raw_dta\auto.dta", clear
    • Stata-formatted datasets use the .dta file extension. Stata can import other file types, but the use command is for .dta files.
    • While it is good practice to include the file extension when importing or exporting data or other files in Stata, it is not strictly necessary. For example, if you type use [datasetname] Stata assumes a .dta file extension.
      use "raw_dta/auto", clear
    • Sometimes when you are working with large datasets you don't want to load the whole dataset into memory.
      • Suppose you only want some of the variables. Let's load auto.dta with only the make, price, and foreign.
            use make price foreign using "C:\Users\michael.baker\Documents\ss368\raw_dta\auto.dta", clear
      • Other times, you are going to restrict the data in some way immediately after importing. You can combine this restriction with the use command. This example imports all the variables, only keeps the rows for foreign cars
            use * using "C:\Users\michael.baker\Documents\ss368\raw_dta\auto.dta" if foreign == 1, clear 

    * limiting the dataset to certain variables and importing only foreign cars 
    use make price foreign using "C:\Users\michael.baker\Documents\ss368\raw_dta\auto.dta" if foreign == 1, clear

  1. Variable window
    • Column/variable names
    • Variable labels. Oftentimes data will not have useful labels. If you want your data to be understood by others, you can label it
      label var foreign "indicator: foreign cars"

  1. Properties window

    • Variables: information on each of the variables
    • Data: information on the dataset loaded in memory in the current dataframe

  2. History

    • Clicking on any of the commands in history will put that command in the command window
    • You can select multiple commands from history and export to a .do file
    • From the command window, you can use page up or page down to navigate between your previously used commands

  3. Other useful items

    • The describe command can be especially helpful to understand your data

    • One of the first things you will do after opening a new dataset is just open it up and look at it. You can do this in Stata using the tool bar or the command window. For the command window:

      browse 

* to look only at specific variables, you can type 
browse make price mpg

    • Basic data types: string (text), numeric, numeric (labeled)

      • The way data is displayed in Stata is based on the format. It is not necessarily how Stata stores the values.

    • Missing values: . for numeric variables, empty for string variables

    • The last important point is that Stata has robust documentation that you should absolutely use. For example, say we wanted to understand how to load data, but we weren't sure of the syntax. We could type

      help use

      to access the full documentation on the use command. The Stata help files should be your first stop any time you are having issues with syntax.

Final Comment: There are often many ways to do the same thing in Stata, and this is true in coding more generally. Sometimes we can say one method is better than another, maybe because it is faster, or uses fewer resources, and requires fewer steps, but oftentimes it is just a matter of preference.
